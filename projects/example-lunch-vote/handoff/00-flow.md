## 대상 뷰포트

- **모바일 우선(mobile-first)** — 점심 정하기는 대부분 폰에서 이뤄진다(PRD §8).
- 데스크톱은 반응형으로 대응(중앙 정렬 또는 full-width).
- 다크모드 지원(시스템 연동 + 수동 토글).

## hi-fi 화면

| 화면 | 역할 |
| --- | --- |
| index.html | 진입점. 사내 계정 로그인 → 홈으로 이동 |
| home.html | 홈. 진행 중 투표 목록(카드), "투표 만들기" 버튼, 오늘의 확정 결과 요약, 빈 상태 안내 |
| create.html | 투표 생성. 제목 입력, 후보 동적 추가/삭제, 마감 시간 선택, 유효성 검사 |
| vote.html | 투표 참여 + 결과. 후보 선택, 실시간 득표 현황, 남은 시간·내 표 표시, 마감 후 최종 순위·동점 안내로 전환 |

## 흐름

- index → home (로그인 성공)
- home → create → vote (투표 개설 후 상세로)
- home → vote (진행 중 투표 참여)
- vote → vote (마감 도달 시 참여 화면이 결과 확정 화면으로 전환)
- create → home (생성 취소·뒤로)
