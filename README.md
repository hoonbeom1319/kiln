# 🔥 Kiln

> 아이디어 한 줄을 넣으면 **기획(PRD) → 디자인 산출물 → 개발 인계 패키지**까지 구워내는 웹 서비스.
>
> *Fire an idea into a dev-ready design hand-off — in your browser, with the model you choose.*

**Kiln**(가마)은 로컬 하네스 [Atelier](../atelier)를 웹·SaaS로 옮긴 프로덕트다. atelier에서 *빚고*, Kiln에서 *구워 완성품을 낸다*. 터미널·설치 없이 브라우저에서, 여러 사용자에게, 모델을 갈아끼울 수 있게 같은 공정을 서비스한다.

여기서 나오는 건 "최종 앱 코드"가 아니라 **개발에 넘길 기획·디자인 산출물**이다.

> 🚧 **초기 개발 중.** 아직 제품이 아니라 뼈대를 세우는 단계다. 진행상황·결정은 **[docs/DECISIONS.md](docs/DECISIONS.md)**.

## ⚡ 하는 일

아이디어 한 줄에서 시작 1회 질문에 답하면, 그 뒤로 사람 게이트 없이 인계 패키지까지 구워낸다.

```
아이디어 ─시작 1회 질문─▶ PRD ─▶ 와이어 ─▶ 검증 ─▶ 디자인 시스템 ─▶ 하이파이 ─▶ 트렌드 감정 ─▶ handoff
                         └────────── 사람은 시작에서 한 번만. 나머지는 자동 게이트가 품질 바닥을 깐다 ──────────┘
```

완성본은 **인계 패키지**(`hand-off.md` + 비주얼 문서·토큰 매핑표·컴포넌트 인벤토리)로 묶여, 받는 쪽 개발자(또는 그쪽 AI)가 **자기 컨벤션으로** 구현한다 — 특정 프레임워크·아키텍처에 안 묶인다.

## 🎯 Atelier와 무엇이 다른가

| | Atelier (원본) | Kiln (이 프로젝트) |
|---|---|---|
| **어디서** | 터미널 (Claude Code) | 브라우저 (웹 SaaS) |
| **누가** | 나 혼자, 로컬 | 여러 사용자, 각자 프로젝트 |
| **모델** | Claude (Opus) 고정 | **교체 가능** — Gemini 우선, 단계별 스왑 |
| **설치** | Node·Playwright·Claude Code | 없음 — 접속만 |

> Kiln은 atelier의 **자동 게이트**(PRD 린트·Playwright 내비/기능 검증·a11y·handoff 완결성)와 **산출물 규격**(독립 실행 HTML·2단 토큰·self-contained handoff)을 그대로 물려받는다. 바뀌는 건 그걸 **누가·어디서·어떤 모델로** 돌리느냐다.

---

## 🧭 왜 Kiln인가

- **설치 장벽 0** — 터미널도 API 키도 없이, 아이디어만 있으면 디자인 인계 패키지가 나온다.
- **모델 자유** — 특정 모델에 안 묶인다. 단계마다 최적 모델을 갈아끼운다.
- **검증이 코드로 박혀 있다** — "예뻐 보인다"가 아니라 클릭되고·상태가 변하고·접근성 통과하고·인계가 완결된 걸 자동으로 강제한다.

## 📦 결과물

받는 쪽에 **그 폴더 하나만 떼서 넘겨도 열리는** self-contained 인계 패키지:

- `hand-off.md` — 붙여넣는 인계 브리프(참고 파일·지킬 계약·만들 순서)
- 비주얼 문서 · 토큰 매핑표 · 컴포넌트 인벤토리
- PRD(DB 설계 근거) · 흐름맵 · 하이파이 화면 · 디자인 토큰
